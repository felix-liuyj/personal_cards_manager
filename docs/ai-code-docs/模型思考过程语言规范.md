# 模型思考过程语言规范

## 核心规则

**模型的所有思考过程必须使用中文**

## 适用范围

- 所有内部思考和推理过程
- 问题分析和解决方案构思
- 代码逻辑设计和架构思考
- 错误诊断和故障排除思路
- 任务规划和执行策略

## 实施要求

1. **思考过程**：使用中文进行逻辑推理和问题分析
2. **代码注释**：保持原有语言风格（英文注释保持英文）
3. **用户交流**：使用中文回复和说明
4. **日志输出**：代码中的日志信息使用中文
5. **文档编写**：技术文档和说明使用中文
6. **注释保留**：修改代码时默认保留原有注释，除非注释已失效或与代码不符
7. **表达风格**：严禁在回复、文档、提交示例和新增注释中使用任何 Emoji 符号
8. **图标约束**：严禁把 Emoji 当作图标、状态标记、按钮内容或占位符使用

## 例外情况

- 代码中的变量名、函数名、类名等标识符使用英文
- API 调用和第三方库接口保持原有英文
- 技术术语和专有名词可保留英文原词
- 引用外部文档或代码时可保持原文

## 质量要求

- 思考过程要清晰、有条理
- 逻辑推理要严谨、完整
- 问题分析要深入、准确
- 解决方案要可行、高效

## 检查标准

- 是否所有思考过程都使用中文
- 逻辑推理是否清晰连贯
- 问题分析是否全面深入
- 解决方案是否切实可行

## 记忆更新

此规则应作为全局规范，适用于所有后续的思考和推理过程。
