# 通用开发与 AI 协作规范

## 适用范围

- 适用于当前工作区内所有 AI 辅助开发、文档编写、代码修改与日常协作
- 作用范围覆盖：
  - `backend/`
  - `admin/`
  - `minigram/`

---

## AI 协作规则

- 思考过程、分析过程、答复说明统一使用中文
- 代码标识符、第三方 API、技术术语保留英文
- 代码注释保持原有语言风格，不主动整批切换注释语言
- 修改代码时默认保留原有注释；若注释已失效、与实现不一致，再一并修正
- 严禁使用任何 Emoji 符号，包括回复、文档、代码注释新增内容、页面文案、占位文案、提交示例和演示内容
- 严禁把 Emoji 当作图标、状态标记、按钮内容或占位符使用
- 涉及前端 icon 时，统一使用内嵌 SVG；具体实现以后续端内规范为准
- 文档、说明、排查结论优先写成可直接执行或可直接核对的形式

## 通用代码风格

- 禁止 `import *`
- 保持方法短小，单个方法有效代码优先控制在 20 行以内
- 优先从具体子模块导入，而不是从模糊的顶层聚合导入

## 命名规则

命名不做全局一刀切，按子项目既有约定执行：

- 后端 Python 文件：`snake_case`
- 管理端 React 组件：`PascalCase`
- 管理端函数：`camelCase`
- 小程序页面目录：`kebab-case`
- 类名：`PascalCase`
- 常量：`UPPER_SNAKE_CASE`

## 文件与脚本规则

- 临时调试脚本、一次性迁移脚本、临时导出脚本不允许直接并入 Git
- 需要长期保留的工具脚本、调试脚本、辅助校验脚本，应放到对应项目的 `tests/` 或明确的工具目录中
- 新增脚本前，先确认仓库内是否已有可复用脚本，不要重复造轮子

## 规则优先级

- 端内专属规则优先于本通用规则
- 当前代码现状优先于抽象模板规则
- 若通用规则与端内既有命名冲突，以对应端的规范文档为准
