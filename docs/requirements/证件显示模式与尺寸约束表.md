# 一、显示模式定义

先统一约束几个显示模式，后续所有证件都按这几类映射。

## 1. `card_horizontal`

横向卡片模式。
适用于大多数身份证、驾照、居住证、社保卡、BRP、州证件等。

### 建议比例

- 标准比例：`1.58 : 1`
- 可选范围：`1.50 ~ 1.65 : 1`

### 使用场景

- 列表页摘要卡
- 详情页顶部主卡

---

## 2. `card_vertical`

竖向卡片模式。
适用于少量需要更强调纵向布局的证件，或产品想做证件差异化展示时使用。

### 建议比例

- 标准比例：`0.63 : 1`
- 可选范围：`0.60 ~ 0.70 : 1`

### 使用场景

- 特殊详情页主卡
- 个别国家/类型的替代展示

---

## 3. `passport_cover`

护照封面摘要模式。
用于把护照以“单本证件”方式数字化展示，不模拟翻开。

### 建议比例

- 标准比例：`1.38 : 1`
- 可选范围：`1.32 ~ 1.45 : 1`

### 使用场景

- 列表页摘要
- 首页摘要
- 详情页顶部主卡

---

## 4. `passport_spread`

护照资料页展开模式。
用于展示资料页或双页展开信息，不作为默认列表形态。

### 建议比例

- 标准比例：`2.0 : 1`
- 可选范围：`1.9 ~ 2.15 : 1`

### 使用场景

- 详情页二级视图
- 展开查看
- 全屏核对页

---

## 5. `document_long`

长条型文档模式。
适用于比例明显偏长的特殊凭证。

### 建议比例

- 标准比例：`2.4 : 1`
- 可选范围：`2.2 ~ 2.8 : 1`

### 使用场景

- 特殊证件详情页
- 少见票据式或凭证式文件

---

## 6. `full_screen_document`

全屏文档模式。
用于高密度信息核对，不强调卡片感。

### 建议比例

- 不固定
- 按屏幕适配

### 使用场景

- 护照资料页
- 需要逐项核对的大字段证件页

---

# 二、尺寸级别定义

所有证件统一采用以下显示级别：

## 1. `compact`

缩略摘要卡
用于：

- 首页
- 列表页
- 搜索结果页

## 2. `standard`

标准主卡
用于：

- 详情页顶部主卡

## 3. `expanded`

展开预览
用于：

- 护照资料页
- 高密度信息页

## 4. `fullscreen`

全屏查看
用于：

- 护照展开视图
- 资料核对场景

# 七、产品层面的统一约束建议

## 1. 列表页统一规则

- 所有非护照类证件，统一按 `card_horizontal`
- 所有护照，统一按 `passport_cover`
- 列表页不展示 `passport_spread`

## 2. 详情页顶部统一规则

- 所有证件默认都以 `standard` 级别显示
- 非护照类：直接展示主卡
- 护照类：展示摘要主卡，不默认展开

## 3. 护照展开规则

- 只在详情页中提供“查看资料页”
- 资料页使用 `passport_spread`
- 若字段较多，支持进入 `full_screen_document`

## 4. 比例优先于绝对尺寸

- 页面实现时优先按比例布局
- 再由容器决定实际像素大小
- 不为不同国家硬编码固定宽高

---

# 八、最终建议

若希望后续继续扩展更多国家，建议把模型固化成：

- **国家**
- **证件类型**
- **摘要显示模式**
- **摘要比例**
- **是否支持展开**
- **展开显示模式**
- **展开比例**
- **是否支持全屏**

这样新增法国、德国、日本、新加坡等国家证件时，只需继续往这张映射表中补充即可，不需要重做整体体系。
